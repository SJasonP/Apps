# Design Direction

## Visual Goal

The site should feel like a polished independent developer app hub.

The design should be:

- Clear
- Trustworthy
- Lightweight
- App-focused
- Suitable for App Store review and regular users

It should not feel like:

- A generic SaaS marketing template
- A blog
- A corporate homepage
- A decorative portfolio site where app information is hard to find

## Layout Principles

- Prioritize app icons, screenshots, app names, and download actions.
- Keep the home page as an app index.
- Keep product pages easy to scan.
- Make support and privacy pages plain and readable.
- Avoid hiding important content behind animations or decorative sections.

## Tone

Copy should be direct and factual.

Prefer:

```text
History Lib helps you browse, preserve, and search clipboard history on macOS.
```

Avoid:

```text
Supercharge your productivity with the ultimate next-generation workflow experience.
```

## Visual Style

Preferred direction:

- Light theme first, with dark mode support if practical
- Use the finalized Liquid Glass Apps icon as the site mark.
- Provide separate light and dark site mark assets, selected with `prefers-color-scheme`.
- Neutral background
- High-quality app screenshots
- Clear typography
- Subtle borders and spacing
- Small, purposeful motion only when it improves interaction

Avoid:

- Heavy gradients
- Oversized generic hero sections
- Decorative graphics unrelated to the apps
- Stock-photo style imagery
- Overly dense dashboards

## First-Version Constraint

The first version should look finished, but it does not need complex interactivity.

Information quality and trustworthy structure are more important than visual novelty.
